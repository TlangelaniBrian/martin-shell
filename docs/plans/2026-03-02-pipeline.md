# CI/CD Pipeline Implementation Plan

> **For Claude:** REQUIRED SUB-SKILL: Use superpowers:executing-plans to implement this plan task-by-task.

**Goal:** Ship a GitHub Actions CI workflow (type-check, lint, build) plus a Vercel project wired to the GitHub repo for automatic production and preview deployments.

**Architecture:** GitHub Actions runs three checks on every push and PR — fast feedback loop. Vercel's native GitHub integration handles all deployments independently: production on `main`, preview URLs on PR branches. A `vercel.json` configures SPA fallback routing so deep links work.

**Tech Stack:** pnpm 10.26.2, Node 20, Rsbuild, vue-tsc, ESLint, Vercel (Other framework preset)

---

### Task 1: GitHub Actions CI workflow

**Files:**
- Create: `.github/workflows/ci.yml`

**Step 1: Create the workflows directory and file**

```bash
mkdir -p .github/workflows
```

**Step 2: Write the workflow**

```yaml
# .github/workflows/ci.yml
name: CI

on:
  push:
    branches: [main]
  pull_request:
    branches: [main]

jobs:
  ci:
    runs-on: ubuntu-latest

    steps:
      - uses: actions/checkout@v4

      - uses: pnpm/action-setup@v4
        with:
          version: 10.26.2

      - uses: actions/setup-node@v4
        with:
          node-version: '20'
          cache: 'pnpm'

      - name: Install dependencies
        run: pnpm install --frozen-lockfile

      - name: Type check
        run: pnpm type-check

      - name: Lint
        run: pnpm lint

      - name: Build
        run: pnpm build
        env:
          VITE_API_URL: https://api.example.com
          WORKSPACE_URL: https://workspace.example.com
```

**Step 3: Validate the YAML is well-formed**

```bash
npx js-yaml .github/workflows/ci.yml
```

Expected: prints the parsed object with no errors.

**Step 4: Commit**

```bash
git add .github/workflows/ci.yml
git commit -m "ci: add GitHub Actions CI workflow (type-check, lint, build)"
```

### Task 2: vercel.json — SPA routing

Vercel serves static files by default. Without a rewrite rule, navigating to `/search` directly (or refreshing) returns a 404. This file fixes that by sending all paths to `index.html`.

**Files:**
- Create: `vercel.json`

**Step 1: Write the file**

```json
{
  "rewrites": [
    { "source": "/(.*)", "destination": "/index.html" }
  ]
}
```

**Step 2: Commit**

```bash
git add vercel.json
git commit -m "chore: add vercel.json for SPA client-side routing"
```

**Step 3: Push to GitHub**

```bash
git push origin main
```

After pushing, open GitHub → Actions tab and verify the CI workflow runs and all three steps pass (type-check, lint, build).

### Task 3: Create the Vercel project

This task is manual — done via the Vercel dashboard.

**Step 1: Import the repo**

1. Go to https://vercel.com/new
2. Select "Continue with GitHub"
3. Find and select `TlangelaniBrian/martin-shell`
4. Click **Import**

**Step 2: Configure build settings**

On the "Configure Project" screen:

| Field | Value |
|---|---|
| Project Name | `martin-shell` |
| Framework Preset | **Other** |
| Root Directory | `.` (leave blank / repo root) |
| Build Command | `pnpm --filter @martin/shell build` |
| Output Directory | `apps/shell/dist` |
| Install Command | `pnpm install --frozen-lockfile` |

**Step 3: Add environment variables**

Still on the same screen, expand "Environment Variables" and add:

| Name | Value | Environments |
|---|---|---|
| `VITE_API_URL` | `https://api.example.com` | Production, Preview, Development |
| `WORKSPACE_URL` | `https://workspace.example.com` | Production, Preview, Development |

> These are placeholder values. Update them in Vercel dashboard → Project Settings → Environment Variables once the real backend URLs are known.

**Step 4: Deploy**

Click **Deploy**. Vercel will:
1. Clone the repo
2. Run `pnpm install --frozen-lockfile`
3. Run `pnpm --filter @martin/shell build`
4. Serve `apps/shell/dist` on a `.vercel.app` domain

Wait for the deployment to show **Ready** (green tick).

**Step 5: Verify SPA routing**

Open the deployed URL. Navigate to `/sign-in` directly in the browser address bar.

Expected: the sign-in page loads (not a 404). If you get a 404, confirm `vercel.json` was included in the push.

**Step 6: Verify preview deploys**

Create a throwaway branch, push a trivial commit, open a PR.

```bash
git checkout -b test/vercel-preview
git commit --allow-empty -m "test: trigger preview deploy"
git push origin test/vercel-preview
```

Open the PR on GitHub. Within ~60 seconds, Vercel should post a comment with a preview URL. Click it and confirm the app loads.

Delete the branch and close the PR when done.

```bash
git checkout main
git branch -d test/vercel-preview
git push origin --delete test/vercel-preview
```
